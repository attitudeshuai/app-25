# TripPacking 功能介绍文档

## 业务背景

结伴旅行时，行李打包往往面临以下痛点：

- **重复携带**：多人各自打包，相同物品（如充电器、洗漱用品、急救包）被多人重复携带，占用行李空间
- **物品遗漏**：靠个人记忆打包，重要物品容易遗漏，到了目的地才发现
- **责任不清**：哪些物品谁负责带没有明确分工，出现遗漏时互相推诿
- **进度不明**：同行伙伴各自打包，不知道整体进度，出发前无法确认是否全部就绪

TripPacking 通过共享清单、物品分配、进度追踪等功能，有效解决以上问题，让结伴旅行的打包过程更加高效、有序。

## 用户角色

| 角色   | 值 | 说明                                   |
|--------|----|----------------------------------------|
| Owner  | 0  | 旅行创建者/管理员，拥有最高操作权限   |
| Member | 1  | 同行成员，可参与清单协作               |

### 角色权限说明

- **Owner**：可编辑/删除旅行、管理所有成员、管理所有物品和分类
- **Member**：可查看旅行信息、编辑分配给自己的物品、标记物品打包状态

## 核心用例流程

```
注册/登录 → 创建旅行 → 邀请成员 → 创建分类 → 添加物品并分配
                                                     ↓
查看统计 ← 标记打包完成 ← 使用模板快速创建 ←───────┘
```

### 用例详情

1. **注册登录**：用户注册账号并登录系统，获取 JWT Token 用于后续 API 调用
2. **创建旅行**：设置旅行标题、目的地、起止日期，创建者自动成为 Owner
3. **邀请成员**：将其他注册用户添加为旅行成员，分配 Owner 或 Member 角色
4. **创建分类**：按衣物、电子设备、洗漱用品、证件等维度创建行李分类
5. **添加物品并分配**：在分类下添加具体物品，设置数量，可分配给指定成员或标记为共享物品，可关联旅行天数
6. **使用模板快速创建**：从公共或个人模板导入预设的分类和物品清单
7. **标记打包完成**：各成员将自己负责的物品标记为已打包，系统实时统计进度
8. **查看统计**：查看全平台打包概览数据和趋势图表

## 模块详细说明

### 1. Auth 认证模块

管理用户身份认证与个人资料。

| 功能     | 说明                                      |
|----------|-------------------------------------------|
| 注册     | 用户名、邮箱、密码注册，邮箱和用户名唯一  |
| 登录     | 邮箱 + 密码登录，返回 JWT Token 和用户信息 |
| 获取个人信息 | 获取当前登录用户的资料                |
| 更新个人信息 | 修改用户名、邮箱、头像                |

### 2. Trips 旅行模块

管理旅行计划的基础信息。

| 功能         | 说明                                              |
|--------------|---------------------------------------------------|
| 创建旅行     | 设置标题、目的地、起止日期，创建者为 Owner        |
| 旅行列表     | 分页查询，支持按关键词、状态筛选                  |
| 我的旅行     | 仅查询当前用户参与的所有旅行                      |
| 旅行详情     | 获取单个旅行的完整信息                            |
| 更新旅行     | 修改旅行标题、目的地、日期等信息（仅 Owner）      |
| 删除旅行     | 删除旅行及其关联的所有数据（仅 Owner）            |
| 更新状态     | Planning(0) / Ongoing(1) / Completed(2)           |

### 3. TripMembers 旅行成员模块

管理旅行参与人员与角色分配。

| 功能         | 说明                                              |
|--------------|---------------------------------------------------|
| 添加成员     | 将用户添加到旅行，指定角色（Owner/Member）        |
| 成员列表     | 按旅行查询成员，支持角色筛选                      |
| 我的成员身份 | 查询当前用户在各旅行中的角色                      |
| 成员详情     | 获取单个成员记录                                  |
| 更新角色     | 修改成员角色（仅 Owner）                          |
| 移除成员     | 将成员从旅行中移除（仅 Owner）                    |

### 4. PackingCategories 行李分类模块

按分类组织行李物品（如衣物、电子设备、证件等）。

| 功能         | 说明                                              |
|--------------|---------------------------------------------------|
| 创建分类     | 指定所属旅行、分类名称、排序权重                  |
| 分类列表     | 按旅行查询分类，支持关键词搜索                    |
| 分类详情     | 获取单个分类信息                                  |
| 更新分类     | 修改分类名称和排序                                |
| 删除分类     | 删除分类（级联删除该分类下所有物品）              |

### 5. PackingItems 行李物品模块

管理具体的打包物品，是系统的核心功能模块。

| 功能         | 说明                                              |
|--------------|---------------------------------------------------|
| 添加物品     | 所属旅行、分类、名称、数量、分配人、是否共享、是否打包、天数关联 |
| 物品列表     | 按旅行、分类、打包状态、是否共享筛选              |
| 物品详情     | 获取单个物品完整信息                              |
| 更新物品     | 修改物品属性，成员仅可修改分配给自己的物品        |
| 删除物品     | 删除物品，Owner 可删除任意物品，Member 仅可删除自己的 |
| 天数关联     | `DayNumber` 字段可关联到旅行第 N 天使用的物品     |

### 6. PackingTemplates 打包模板模块

预设行李清单模板，支持快速创建。

| 功能         | 说明                                              |
|--------------|---------------------------------------------------|
| 创建模板     | 模板名称、分类、物品 JSON 结构、创建人            |
| 模板列表     | 公共列表，无需登录即可浏览，支持分类筛选          |
| 模板详情     | 获取模板完整信息（含物品结构）                    |
| 更新模板     | 修改模板内容（仅创建者）                          |
| 删除模板     | 删除模板（仅创建者）                              |

### 7. Stats 统计模块

提供平台级数据统计与趋势分析。

| 功能         | 说明                                              |
|--------------|---------------------------------------------------|
| 概览统计     | 总用户数、总旅行数、总物品数、已打包数、未打包数、打包进度百分比 |
| 趋势统计     | 按日期范围返回每日创建的旅行数和物品数，支持时间区间查询 |

## ER 关系图

### 实体关系描述

```
Users (1) ────── owns (N) ──────→ Trips
  │                                 │
  │                                 │ has
  │                                 ▼
  │                        PackingCategories (N)
  │                                 │ has
  │                                 ▼
  │                        PackingItems (N)
  │                                 ▲
  │                                 │ assigned
  └─────────────────────────────────┘

Users (N) ←── TripMembers (N) ──→ Trips (N)
  (many-to-many through TripMembers join table)

Users (1) ────── creates (N) ──→ PackingTemplates
```

### 详细关系定义

| 关系                          | 类型        | 说明                          |
|-------------------------------|-------------|-------------------------------|
| Users → Trips                 | 1:N         | 一个用户可创建多个旅行 (Owner)|
| Users ↔ TripMembers ↔ Trips   | M:N         | 用户通过 TripMembers 关联多旅行，旅行有多个成员 |
| Trips → PackingCategories     | 1:N         | 一个旅行有多个行李分类        |
| Trips → PackingItems          | 1:N         | 一个旅行有多个行李物品        |
| PackingCategories → PackingItems | 1:N      | 一个分类下有多个物品          |
| Users → PackingTemplates      | 1:N         | 一个用户可创建多个模板        |
| Users → PackingItems          | 1:N (可选)  | 物品可分配给一个用户 (AssignedTo) |

## 业务规则

1. **旅行管理权限**：只有旅行的 Owner 可以修改或删除旅行信息，普通 Member 只能查看
2. **成员管理权限**：只有 Owner 可以添加、移除成员或修改成员角色
3. **物品编辑权限**：
   - Owner 可编辑旅行中所有物品
   - Member 只能编辑分配给自己的物品（`AssignedTo == 当前用户ID`）
   - 共享物品（`IsShared = true`）可由任意成员标记打包状态
4. **物品删除权限**：Owner 可删除任意物品，Member 仅可删除自己创建或分配给自己的物品
5. **模板权限**：
   - 模板列表公开可读，无需登录
   - 只有模板创建者可以修改或删除模板
6. **唯一性约束**：
   - 用户名唯一
   - 邮箱唯一
   - 同一旅行中同一用户只能有一个成员身份（TripId + UserId 唯一）
7. **状态流转**：旅行状态 Planning(0) → Ongoing(1) → Completed(2)

## API 调用示例

### 示例 1：用户注册

```bash
curl -X POST http://localhost:8095/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "traveler01",
    "email": "traveler01@example.com",
    "password": "password123"
  }'
```

响应：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "user": {
      "id": 1,
      "username": "traveler01",
      "email": "traveler01@example.com",
      "avatar": null
    }
  }
}
```

---

### 示例 2：登录并创建旅行

**第一步：登录获取 Token**

```bash
curl -X POST http://localhost:8095/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "traveler01@example.com",
    "password": "password123"
  }'
```

**第二步：使用 Token 创建旅行**

```bash
curl -X POST http://localhost:8095/api/trips \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your-token-here>" \
  -d '{
    "title": "日本东京 7 日游",
    "destination": "Tokyo, Japan",
    "startDate": "2024-07-01T00:00:00Z",
    "endDate": "2024-07-07T00:00:00Z"
  }'
```

响应：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "ownerId": 1,
    "title": "日本东京 7 日游",
    "destination": "Tokyo, Japan",
    "startDate": "2024-07-01T00:00:00Z",
    "endDate": "2024-07-07T00:00:00Z",
    "status": 0,
    "createdAt": "2024-06-01T12:00:00Z"
  }
}
```

---

    "categoryId": 1,

    "assignedTo": 1,
